# قطعی برق · Outage Tracker

A modern React front-end for displaying planned power-outage events (Mazandaran, Iran).
Designed to be **embedded into your Go backend** via `embed.FS` — single binary, no Node
runtime needed in production.

The React app fetches outage data from `/api/events` (relative URL, same origin as your
Go server), so there is zero CORS configuration. Free, no-key weather data (temperature
+ humidity) is overlaid onto each outage from 3 open providers.

## Features

- 🇮🇷 Full Persian (RTL) UI with the Vazirmatn font
- 🌗 Modern dark UI with gradient accents, glassmorphism, smooth transitions
- 🔎 Live filtering by city, status, search, date range, and sort
- 🌡️ Weather data per outage (temperature + humidity) from 3 free providers:
  - **Open-Meteo** — best historical + forecast coverage (default)
  - **Met.no** — Norwegian Meteorological Institute
  - **7Timer!** — 3-hour step forecasts
- ⚡ Caches weather requests so switching providers is instant
- 📦 **Single self-contained `index.html` build** — embed in Go via `//go:embed`

## Build the React app

```bash
npm install
npm run build     # produces dist/index.html (~198 KB, everything inlined)
```

The build outputs a single `dist/index.html` with all JS, CSS, and SVG inlined.
No external script/style/image files are required (Google Fonts is optional).

## Embed in your Go app

A complete working example lives in `go-example/main.go`. The key parts:

```go
import (
    "embed"
    "io/fs"
    "net/http"
)

//go:embed all:dist
var distFS embed.FS

func main() {
    mux := http.NewServeMux()

    // Your existing outage API
    mux.HandleFunc("/api/events", yourEventsHandler)

    // Embedded React app with SPA fallback
    sub, _ := fs.Sub(distFS, "dist")
    mux.Handle("/", spaHandler(sub))

    http.ListenAndServe(":8080", mux)
}

// spaHandler serves embedded files and falls back to index.html for unknown
// routes (so client-side routing works if you add it later).
```

### Integrate with your existing backend

1. **Copy the `dist/` directory** into your Go project (next to the file
   that has the `//go:embed` directive).
2. **Register the API route** at `/api/events` with your existing handler.
3. **Mount the SPA handler** on `/` (or as a catch-all).
4. **Build your Go binary** — the React app is now part of it.

```
your-go-app/
├── main.go              ← //go:embed all:dist
├── dist/
│   └── index.html       ← generated by `npm run build`
└── go.mod
```

That's it. No CORS, no separate frontend server, no Node in production.

## Run the Go example

```bash
# 1. Build the React app
npm run build

# 2. Copy dist into the Go example (or do this in your own project)
cp dist/index.html go-example/dist/

# 3. Build and run the Go server
cd go-example
go build -o server .
./server
# → open http://127.0.0.1:8080
```

## Dev workflow (no Go integration needed)

```bash
npm run dev          # http://localhost:5173 with /api/events proxy
# OR
npm run build && node serve.js   # http://127.0.0.1:4175
```

If your Go server isn't running on `:8080` yet, you can run a local mock:

```bash
node mock-server.js  # serves sample data on :8081
```

## Notes on weather data

- **Open-Meteo**: queries `geocoding-api.open-meteo.com` to resolve the city,
  then `archive-api.open-meteo.com` for past outages or `api.open-meteo.com`
  for upcoming ones. Supports hourly resolution.
- **Met.no**: requires coordinates; returns ~3-day forecasts only. Past outages
  may have no data.
- **7Timer!**: returns 3-hour step forecasts (~7 days). Its `rh2m` field is a
  humidity index, not standard % RH — treat it as a relative indicator.

All three providers require no API key. The frontend talks to them directly
from the browser; nothing is proxied or stored.
